# Release Notes

This directory tracks version-specific changes and known issues for this fork.

## File naming

- Use one file per released or testable version.
- Prefer names like `v2.1.87.md`.

## Suggested sections

- `## Changes`: user-visible changes, new commands, behavior updates, important fixes.
- `## Known Issues`: confirmed remaining issues, limitations, or follow-up risks.
- `## Verification`: optional notes about build checks or manual validation.
