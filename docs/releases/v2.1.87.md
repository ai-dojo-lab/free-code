# v2.1.87

Date: 2026-04-01

## Changes

- Added ChatGPT OAuth login support with `/login-chatgpt`.
- Added `/logout-chatgpt` to clear only ChatGPT login state without logging out of Anthropic.
- Added GPT/Codex model discovery and session model selection after ChatGPT login.
- Routed GPT/Codex requests through the ChatGPT Codex backend instead of the standard OpenAI API key flow.
- Added GPT/OpenAI cost visibility in `/cost`, `/status`, status bar, and `/stats`.
- Added README documentation for ChatGPT login and usage.
- Added a Bun-native `ripgrep` fallback for compiled builds where vendored `rg` is not present on disk.

## Fixes

- Fixed model picker cancel behavior so canceling GPT model selection no longer auto-selects the first GPT model.
- Fixed ChatGPT logout cleanup so previously selected GPT models do not remain in `/model` after logout.
- Fixed GPT model validation so stale validation cache does not survive logout.
- Fixed token refresh handling to preserve the previous refresh token when the refresh response omits a new one.
- Fixed OAuth refresh scope to match the original ChatGPT login scope.
- Fixed dynamic GPT/Codex model pricing fallback so models like `gpt-5.3-codex` do not fall back to Claude pricing.
- Improved OpenAI/Codex SSE handling for malformed frames, completion-only responses, and finish-reason mapping.

## Known Issues

- No confirmed blocking bugs remain in the ChatGPT login, model switch, and logout main flow after current validation.
- GPT multimodal input behavior has not been fully regression-tested.
- Long-running real-world token refresh behavior has not been soak-tested over extended sessions.
- Upstream ChatGPT Codex backend event format changes may still require future parser updates.

## Verification

- `bun run build:dev:full`
- Manual checks completed for `/login-chatgpt`, GPT model selection, `/model`, and `/logout-chatgpt`
